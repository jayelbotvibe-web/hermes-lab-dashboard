Hermes Lab — Boot to Workflow

What happens from power-on → ready to pentest/forensics
PHASE 1 — Automatic on Boot
1
[AUTO] Ubuntu boots. Network comes up.
2
[AUTO] systemd starts hermes-sift-vm.service
→ vmrun start SIFT.vmx nogui
→ SIFT VM boots in background (45s to SSH-ready)
3
[AUTO] ~/.config/autostart/ launches hermes-tray
→ System tray icon appears (top-right, near clock)
4
[AUTO] Docker daemon starts. No containers yet (restart policy removed).
Tray shows: Pentest: ○ OFF  |  Forensics: ○ OFF
PHASE 2 — You Decide
🛡️ PENTEST PATH
5a
[CLICK] Tray → ▶ Start Pentest
Terminal opens, runs pentest
6a
[AUTO] mount-pentest-vault
→ cryptsetup open with ~/.pentest-keyfile
→ LUKS mounted at ~/pentest
No passphrase needed
7a
[AUTO] pentest-up.sh
→ docker compose up -d (web + net + osint)
→ Tools pre-installed in images (no apt install)
→ ✓ kali-web ✓ kali-net ✓ osint-tools
8a
[YOU] Open Hermes Desktop
→ Switch to pentest profile (bottom-left)
→ Start new chat
→ /skill pentest-engage
🔍 FORENSICS PATH
5b
[CLICK] Tray → ▶ Start Forensics
Terminal opens, runs forensics
6b
[AUTO] forensics-up.sh
→ LUKS mounts with ~/.forensics-keyfile
→ SIFT VM already running (from boot)
→ Waits for SSH on sansforensics@172.16.146.128
7b
[AUTO] session-canary.sh
→ Validates all 12 forensic tools
→ ✓ Docker tools ✓ SIFT tools
8b
[YOU] Open Hermes Desktop
→ Switch to forensics profile
→ Start new chat
PHASE 3 — Shutdown
9
[CLICK] Tray → ■ Stop Pentest / ■ Stop Forensics
→ pentest-down.sh stops containers + unmounts LUKS
→ forensics-down.sh stops SIFT VM + unmounts LUKS
10
[AUTO] Everything encrypted at rest. Safe to power off.
THINGS GO WRONG?
📋 View Log → tray menu → opens today's startup log
🔧 Repair → pentest-repair.sh — diagnose + auto-fix
📖 Troubleshooting → cat ~/pentest-repo/docs/TROUBLESHOOTING.md
🧠 Skill → /skill pentest-ops in any Hermes chat
