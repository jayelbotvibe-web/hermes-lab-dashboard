Hermes Profile Manager — Supervisor Architecture

Pattern: Supervisor (Orchestrator-Worker) • Dispatch: hermes -z --profile • No profile changes required

What Changes (Nothing)

  • • Forensics profile — UNCHANGED (SOUL, 10 skills, Docker images, scripts)
  • • Pentest profile — UNCHANGED (SOUL, pentest skills, containers, VPN)
  • • Config files — UNCHANGED (model, cwd, terminal backend, approvals)
  • • Memory files — UNCHANGED (each profile keeps its own)
  • • Session databases — UNCHANGED (independent, searchable cross-profile)

What Changes (New Layer)

  • • Default profile takes conscious Manager role (route + dispatch + validate)
  • • Dispatch via: hermes -z "[prompt]" --profile [forensics|pentest]
  • • Pre-flight health checks: check-forensics-health.sh / check-pentest-health.sh
  • • Dispatch audit logs: ~/.hermes/logs/dispatch-*.log
  • • Manager skill: lab-manager (SKILL.md + RUNBOOK.md)

Pattern: Supervisor (Orchestrator-Worker)

  • • Industry standard — used by Azure, Anthropic, LangGraph, CrewAI
  • • Manager is thin: route, health check, construct prompt, validate, present
  • • Workers are autonomous: own SOUL, skills, memories, toolchains
  • • Inter-agent via shared filesystem + session_search() cross-profile
  • • Fallback: hermes mcp serve --profile forensics (MCP bridge pattern)
